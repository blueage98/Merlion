# 웹앱 정적/동적 구조 분석 프롬프트

이 웹앱의 구조를 **정적(static)** 요소와 **동적(dynamic)** 요소로 나누어 분석해줘.

## (1) 정적 구조
- 빌드 시점에 고정되는 자산: HTML 템플릿, CSS, 이미지, 폰트, 번들된 JS/정적 자산 디렉터리 위치
- 정적 라우트/페이지 정의: URL이 요청 시점 데이터와 무관하게 고정된 콘텐츠를 반환하는 곳
- 설정 파일: 환경변수, 빌드 설정, 라우팅 테이블처럼 배포 전에 결정되는 값
- 정적 자산이 서빙되는 방식(CDN, 앱 서버의 static 폴더, 별도 스토리지 등)

## (2) 동적 구조
- 요청마다 값이 바뀌는 부분: 서버 사이드 렌더링/템플릿에 주입되는 데이터, API 응답
- 상태 관리: 세션/스토어(예: 서버 세션, 클라이언트 상태 저장소, 캐시)가 어디서 어떻게 유지되는지
- 사용자 입력을 받아 처리하는 진입점(라우트, 컨트롤러, 콜백, 핸들러 등 프레임워크에 맞는 용어로)과 그것이 호출하는 로직/서비스 계층
- 데이터베이스 또는 영속 계층과의 상호작용 지점(쿼리, ORM 호출, 파일 I/O 등)
- 실시간/비동기 요소: 웹소켓, 폴링, 백그라운드 잡, 캐시 무효화 트리거

## (3) 경계와 연결
- 정적 자산이 동적 로직과 만나는 지점(예: 정적 페이지에 동적 데이터를 주입하는 템플릿 렌더링 지점)
- 빌드 타임 vs 런타임 경계가 어디서 그어지는지 (예: SSG/SSR/CSR 중 무엇을 쓰는지, 어떤 프레임워크 관례를 따르는지)
- 캐싱 계층이 정적/동적 어느 쪽에 속하는지, 캐시 무효화는 무엇이 트리거하는지

## 출력 형식
각 항목에 대해 실제 파일 경로와 코드 위치(`file:line`)를 근거로 제시하고, 프레임워크에 특화된 실제 이름(라우트 파일명, 컴포넌트명, 함수명)을 사용해서 설명해줘. 이 리포에 해당 계층이 존재하지 않으면 "없음"이라고 명시하고 대체 구조가 있다면 함께 제시해줘.
